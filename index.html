Core Features

本地文件与挂载目录

支持 App 沙盒 Documents，也支持通过系统文档选择器挂载外部目录并持久保存授权。

  • 统一文件列表、目录导航、搜索与排序
  • 属性面板、上下文菜单与侧滑操作
  • 适合整理文稿、项目文件和下载内容

远程服务器连接

通过同一套远程文件抽象接入多种后端，降低不同协议之间的使用心智成本。

  • AList、WebDAV、SFTP、SMB 已接入
  • 远程目录浏览、下载、上传、重命名、复制与移动
  • 第一个服务器免费可用，更多服务器可通过会员或激励广告解锁

预览、播放与阅读

针对不同文件类型选择更合适的展示方式，而不是只依赖单一系统预览。

  • 图片浏览、视频播放、音乐播放器
  • PDF 阅读、标记、编辑与导出
  • Office、压缩包和其它文档预览

文本编辑与对比

既能直接编辑常见文本文件，也能进行文件级与文本级差异比较。

  • 支持代码和配置文本的语法高亮
  • 左右对比、上下对比、统一视图
  • 差异统计、汇总页与可配置忽略规则

工具箱

把高频但不适合塞进列表工具栏的功能集中到工具箱入口。

  • 文件夹对比、文件对比、文本比较
  • 文本生成 PDF、多图生成 PDF、合并 PDF
  • 创建加密容器、挂载加密容器

会员与激励广告

会员主打去广告、无限服务器与永久解锁高级工具；激励广告提供 24 小时临时解锁。

  • 安装后前 7 天默认免广告且可使用全部功能
  • 已购会员不显示横幅广告与应用内广告
  • 高级功能入口统一走同一套权限判断

Protocols & Sources

本地和远程都放在一个视角里处理

对于普通用户，重要的不是协议名本身，而是这些来源都能被一致地浏览、预览、下载和管理。

AList

适合接入网盘聚合与个人存储。音频与视频可按需解析直链，图片可直接进入预览链路。

WebDAV

支持目录读取、上传下载、复制移动与视频本地代理播放，适合 NAS 和私有云文件管理。

SFTP / SMB

兼顾远程目录管理与媒体访问，通过应用内本地 HTTP 代理处理播放器流式读取。

本地 / 挂载目录

既可直接管理 App 文档，也能访问用户授权的外部目录，适合与系统“文件”App 搭配工作。

统一能力

  • 浏览目录与文件属性
  • 新建文件夹、重命名、删除
  • 上传、下载、复制、移动
  • 图片、视频、PDF、文本预览
  • 播放器流式访问与缓存策略

当前状态

README 中已经明确记录了不同文件类型在本地、AList、WebDAV、SFTP、SMB 下的实际访问方式， 包括直连、代理和先落盘缓存三类路径。

FAQ

常见问题

这个网站里的内容和 App 当前实现一致吗？

站点文案主要整理自当前仓库的 README、设置页、工具箱、服务器列表和会员相关代码，尽量与代码现状一致，而不是只写理想功能。

是否支持远程视频和音乐播放？

支持。当前实现会根据来源使用直链或本地代理方式，把 WebDAV、SFTP、SMB 等远程流转换成更稳定的播放路径。

免费用户能用到什么程度？

基础本地文件管理、首个服务器、音视频播放、PDF 阅读与基础预览都能直接使用；更多服务器和部分高级工具需要会员或激励广告解锁。

适合发布到 GitHub Pages 吗？

可以。这个目录使用纯静态 HTML、CSS 和少量原生 JavaScript，无构建步骤，直接用 GitHub Pages 指向 `docs` 目录即可。

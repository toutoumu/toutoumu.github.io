核心功能

围绕真实文件任务，而不是只做一个“能看目录”的 App

官网文案基于应用当前功能整理而来，重点强调你在日常工作中真正会反复用到的几类能力。

统一管理不同来源

同时管理 App 本地文件、通过系统授权挂载的外部目录，以及多种远程服务器来源。

  • 统一目录浏览、搜索、排序和属性查看
  • 本地与远程共用一致的文件操作逻辑
  • 适合把“手机文件 + NAS + 服务器”放在一个入口里处理

打开和预览更多文件类型

面向实际内容消费场景，支持图片、视频、音频、PDF、文本、压缩包以及常见文档类型预览。

  • 远程媒体可直接播放，不必都先下载到本地
  • 文本可直接阅读与编辑
  • 文档类文件可快速进入系统预览链路

跨来源传输与整理

不是单向下载器，而是一套完整的文件移动、复制、上传和导入工作流。

  • 本地到远程上传，远程到本地下载
  • 同一界面中完成重命名、新建文件夹、删除和批量操作
  • 下载中心可持续跟踪下载状态和历史记录

为内容处理准备的工具箱

除了管理文件，还内置一批面向整理、核对与转换的工具。

  • 文本对比与文件夹对比
  • 文本生成 PDF、多图生成 PDF、合并 PDF
  • 音乐播放器与播放列表管理

本地 Web 上传入口

让同一局域网内的浏览器把文件直接传到你的设备，无需额外搭建服务。

  • 适合临时从电脑传素材到 iPhone 或 iPad
  • 减少 AirDrop 不稳定或聊天工具压缩带来的麻烦
  • 更适合小范围、即时的文件投递

典型场景

从“找文件”到“处理文件”，一条链路尽量少跳转

01

在手机上访问 NAS 或远程目录

连接 SMB、WebDAV、SFTP 或 AList 后，直接浏览文件树，按需预览、下载或播放远程内容。

02

整理本地与远程之间的文件

把图片、文档、音频或压缩包在不同来源间移动、复制、导入、上传，不用来回切换多个 App。

03

快速检查差异和生成文档

对比文本内容、核对目录差异，或者把文本和图片整理成 PDF，适合轻量办公与内容归档。

04

在设备上构建自己的文件中心

通过下载中心、音乐播放列表、本地 Web 上传和加密容器，把 FileZen 变成日常的个人文件入口。

免费与激励

核心思路是尽量让所有功能都能免费用

FileZen 不想做“免费只能看看，真正功能都锁住”的产品。核心能力默认面向所有用户开放， 部分高级功能会在特定阶段通过激励广告提供解锁时段；如果你希望长期无打扰使用，也可以选择会员方案。

所有人都能体验完整能力

  • 文件夹对比、文本对比、PDF 工具都不是永久封死
  • 本地文件管理保持可用，不做基础能力阉割
  • 适合先免费体验，再决定是否长期使用

激励广告负责解锁部分高级场景

  • 部分高级能力会通过激励广告获得时效权限
  • 适合偶尔需要文件夹对比、PDF 工具或更多连接位的用户
  • 让轻度用户也能低门槛用到完整工作流

会员是增强体验，不是唯一入口

  • 适合希望更稳定、长期、少打扰使用的重度用户
  • 可移除广告并减少反复触发激励解锁的频率
  • 如果只是偶尔用高级功能，免费加激励的方式也能覆盖
